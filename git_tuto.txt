#github 사용법

1. github 검색

2. github 회원가입

3. 로그인 하기

4. NEW버튼 클릭하고 repository만들기

5. repo이름쓰고, public쳌 하고 create클릭

6. 자신의 운영체제에 맞는 'git' 다운받으러 ㄱㄱ
ㄴgithub =원격 저장소, git = 본인 컴퓨터(로컬 저장소)에서 github로 보내주는 역할

7. git 다운 다받으면 git bash 혹은 cmd열어서 환경설정 하기
ㄴ윈도우 버튼 + r -> cmd검색

8. cmd창에서 git명령어를 이용해서 환경설정
- git config --global
ㄴ나의 컴퓨터 전체에서 동일한 옵션을 사용하도록하는 명령어

- git config --global user.name "사용자 이름"
ㄴ깃허브 계정 이름

- git config --global user.email "이메일"
ㄴ깃허브 계정 이메일

9. 내 컴퓨터 편안한 위치에 폴더 만들기

10. 내 컴퓨터와 깃허브repo와 동기화 해주기
ㄴcmd창에서 9번에서 만든 폴더로 이동하기
ㄴcd명령어를 사용하여 이동하면 됩니다.
ㄴ git clone "깃허브 repo 주소"     명령어 치기
	     ㄴ이거는 4번에서 만든 repository들어가면 있습니다
※clone 명령어는 자신의 폴더를 깃헙 repo에 복사하는 겁니다.

ㄴgit clone을 하면 아마 비어있는 폴더를 복사한다고 warning메세지가 뜰텐데 9번에서 만든 폴더에 암것도 없어서 그렇습니다. warning메세지는 그냥 무시해 됩니다.

11. 9번에서 만든 폴더로 들어가서 .git 폴더 확인하기
ㄴ내가 만든 폴더로 가보면 깃허브에서 만들었던 repository의 이름으로 폴더하나가 만들어졌을테고, 그 안에 .git 이라는 폴더가 만들어 졌을텐데 이게 '안보이는'이유는 윈도우에서 숨김처리 해서 그렇습니다
ㄴ그래서 '폴더 옵션' -> '보기' -> '알려진 파일 형식의 파일 확장명 숨기기' 체크 풀기하면 .git 폴더가 보일겁니다.
ㄴ여기까지하면 이제 나의 컴퓨터는 로컬 저장소가 되었고 깃헙은 원격 저장소가 되었습니다.

#원격 저장소(깃헙)에 업데이트 하는 법

1. cmd창에서 로컬 저장소 역할을 하는 폴더로 이동
ㄴcd 명령어로 이동하기

2. git add해주기
ㄴgit add "파일 or 폴더 이름"
ㄴ이는 변경 혹은 신규 파일/폴더를 
기록하는 명령어 = 장바구니 담기

3. git commit -m "메모"
ㄴadd해주었던 파일/폴더들을 내 컴퓨터에 변경했다고 기록 남기는 명령어 = 계산대로 가져가기
ㄴ나중에 깃헙에 업뎃되었을때 어떤 것이 업뎃되었는지 혹은 어떤 변경점이 있는지 메모 해줄 수 있다

4. git push
ㄴ로컬 저장소에서 원격 저장소로 보내주는 명령어 = 집에 가져가기
ㄴ여기까지 했으면 깃헙에 접속해서 내가 push한 파일들이 잘 업뎃되었는지 확인해보자

5. git pull
ㄴ원격 저장소에 있는 파일들을 로컬 저장소에 덮어쓰기 하기

※git push오류가 나면 로컬저장소와 원격저장소에 있는 파일들간에 차이가 있어서 push가 안되는 것이다. 그럴때는 원격과 로컬간에 차이를 없애야 정상적으로 push를 할수있기에 먼저 git pull을 사용해 원격을 로컬에 덮어쓰기하고 그다음에 로컬의 변경점을 원격에 push해주면 된다.
